# Game Hat UI Philosophy

## Overview

Game Hat UI should feel:
- lightweight
- social
- readable
- familiar
- fast to understand during live play

The UI should support gameplay rather than distract from it.

---

## Core Principle

The focus should remain on:
- people
- prompts
- interaction
- shared play

UI complexity should stay secondary.

---

## Screen Priorities

Important gameplay information should remain visually prioritized.

Examples:
- current Active Player
- timer
- score
- prompt state
- next required action

---

## Shared Component Philosophy

Game Hat should reuse shared Inner Circle components whenever possible.

Examples:
- dialogs
- buttons
- timers
- cards
- participant lists

Avoid creating unnecessary custom UI systems.

---

## Navigation Philosophy

Game Night and Game Hat should use:
- predictable navigation
- minimal nested complexity
- fast transitions between gameplay states

---

## Multiplayer Clarity

Players should always understand:
- whose turn it is
- what phase the session is in
- whether the timer is active
- whether they are waiting or acting

---

## Mobile-First Design

The primary experience should work comfortably on phones.

Priorities:
- large tap targets
- readable countdowns
- clear prompts
- minimal clutter

---

## Visual Tone

Preferred tone:
- warm
- playful
- modern
- socially comfortable

Avoid:
- overly corporate UI
- excessive visual noise
- hyper-gamified complexity

---

## Session Screens

Likely screens:
- lobby
- prompt submission
- active round
- scoring/results
- session completion

---

## Animation Philosophy

Animations should:
- clarify state changes
- improve readability
- remain lightweight

Avoid:
- excessive transitions
- gameplay delays caused by animation
- unnecessary visual effects

---

## Design Principles

- clarity over spectacle
- familiarity over novelty
- reusable systems over one-off UI
- gameplay-first presentation
