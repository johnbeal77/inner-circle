# Game Hat Content Rules

## Overview

Game Hat uses user-generated prompts.

The platform should support fun, flexible play while maintaining reasonable safety and moderation boundaries.

---

## Core Principle

Prompt systems should prioritize:
- trust within Circles
- player comfort
- low-friction moderation
- predictable behavior

---

## Default Assumption

Inner Circle assumes:
- Circles are trusted groups
- moderation should remain lightweight initially

The MVP should avoid overcomplicated moderation systems.

---

## Disallowed Content (Platform-Level)

Prompts should not contain:
- illegal content
- explicit threats
- targeted harassment
- malicious spam
- platform abuse content

---

## Circle Autonomy

Circles may define their own social expectations.

Future possibilities:
- family-safe Circles
- mature-content-enabled Circles
- custom moderation preferences

---

## Reporting

Possible future features:
- report prompt
- report participant behavior
- Host moderation controls

---

## Host Authority

Hosts may eventually:
- remove prompts
- skip prompts
- remove Players
- cancel sessions

---

## Prompt Visibility

Prompts should remain private until assigned.

Players should not browse the full prompt pool during active gameplay.

---

## AI Moderation (Future)

Possible future moderation assistance:
- profanity detection
- duplicate detection
- unsafe content flagging

AI moderation should assist rather than dominate trusted-group play.

---

## Design Principles

- lightweight moderation
- trust-first design
- predictable rules
- avoid unnecessary friction
- preserve playful social interaction
